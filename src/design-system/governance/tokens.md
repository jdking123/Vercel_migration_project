# Token Governance

## Naming
- Use semantic names (e.g., `--color-accent`, `--space-4`)
- Avoid presentational names (e.g., `--blue`, `--big`)
- Use OKLCH values for all colors

## Versioning
- Tokens follow semantic versioning
- Breaking changes require a major version bump
- Deprecated tokens must remain for one full minor version

## Categories
- Color
- Typography
- Spacing
- Radius
- Shadow
- Layout

## Change Process
1. Proposal submitted
2. Review by design + engineering
3. Approval by governance committee
4. Documented in changelog
