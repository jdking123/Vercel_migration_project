# Component Governance

## Requirements
- Must use tokens
- Must be WCAG 2.2 AA compliant
- Must support dark mode
- Must support multilingual content
- Must use CVA for variants
- Must use forwardRef

## Breaking Changes
- Require major version bump
- Must include migration guide
- Must include deprecation notice

## Review Checklist
- Accessibility
- Responsiveness
- Token usage
- Naming consistency
- API stability
