# Extension Points & Boundaries

## Purpose
Define where and how the design system can be safely extended.

## Extension Points
- Components
- Patterns
- Tokens
- Editorial rules
- Page templates

## Boundaries
- Core components cannot be overridden
- Tokens must remain immutable except via governance
- Extensions must not introduce new global styles
- Extensions must not break WCAG compliance

## Allowed Extensions
- Domain-specific components
- Domain-specific patterns
- Domain-specific tokens (namespaced)
