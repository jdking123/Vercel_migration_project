# Platform Expansion Guidelines

## Purpose
Ensure the design system can scale across platforms.

## Supported Platforms
- Web (Next.js)
- Documentation (MDX)
- Internal tools
- External portals

## Expansion Rules
- Tokens must remain platform-agnostic
- Components must be framework-agnostic where possible
- Patterns must be content-agnostic
- Editorial rules must support multilingual content

## Future Targets
- Mobile
- PDF generation
- Email templates
